# TRIZ（発明的問題解決理論）

## 概要
アルトシュラーが数十万件の特許を分析して導出した、技術的矛盾を体系的に解消するための40の発明原理。

## 核心概念: 技術的矛盾と物理的矛盾

### 技術的矛盾
あるパラメータを改善すると、別のパラメータが悪化する。
例: 強度を上げると重量が増す / 速度を上げると精度が落ちる

### 物理的矛盾
同一のパラメータが、同時に大きくも小さくもある必要がある。
例: 飛行機の車輪は離着陸時には存在し、飛行中には存在しない必要がある → 格納式

## 主要な発明原理（問題解決に特に汎用性が高いもの）

### 原理 1: 分割
- 対象を独立した部分に分割する
- 対象を分解しやすくする
- 分割の度合いを高める
- 適用: モノリシックなシステム → マイクロサービス

### 原理 2: 分離（抽出）
- 対象から必要な部分だけを抽出する
- 適用: 問題の核心だけを取り出し、周辺を無視する

### 原理 10: 先取り作用
- 必要な作用を予め実行しておく
- 対象を予め配置しておく
- 適用: キャッシュ、プリフェッチ、予防的対策

### 原理 13: 逆発想
- 予定されている作用の逆を実行する
- 対象を反転させる
- 適用: 問題を解くのではなく、問題が生じない条件を作る

### 原理 17: 他の次元への移行
- 一次元的な配列を二次元、三次元に変更する
- 対象を傾ける / 裏返す
- 適用: パラドックスの次元上昇（problem-decomposition.md 参照）

### 原理 25: セルフサービス
- 対象が自分自身にサービスするようにする
- 廃棄物やエネルギーを利用する
- 適用: 自己修復システム、自己組織化

### 原理 35: パラメータの変更
- 物理的状態を変える（固体→液体→気体）
- 濃度や密度を変える
- 柔軟性の度合いを変える
- 適用: 固定的な構造を流動的にする

### 原理 40: 複合材料
- 均質な材料を複合材料に変更する
- 適用: 単一アプローチから複合アプローチへ

## 矛盾マトリクスの使い方
1. 改善したいパラメータを特定
2. 悪化するパラメータを特定
3. マトリクスから推奨される発明原理を参照
4. 原理を課題の文脈に翻訳して適用

## 非技術領域への適用
TRIZは本来は工学向けだが、以下のように抽象化して適用可能:
- 「強度」→ システムの頑健性
- 「重量」→ 運用コスト / 複雑性
- 「速度」→ 応答性 / 意思決定速度
- 「温度」→ 組織の緊張度 / ストレスレベル
