# Lateral Thinking（水平思考）

## 概要
エドワード・デ・ボーノが提唱した、論理的・直線的な思考（Vertical Thinking）では到達できない解を、意図的に思考の方向を変えることで発見する手法。

## Vertical Thinking との対比
| Vertical（垂直思考） | Lateral（水平思考） |
|---|---|
| 正しさを求める | 豊かさを求める |
| 1歩ずつ論理的に | 飛躍的に、非連続に |
| 関係あるものだけを見る | 関係なさそうなものも見る |
| 最も有望な方向に進む | 最も意外な方向に進む |
| 結論に向かう | 出発点を変える |

## 主要テクニック

### 1. ランダムエントリー（Random Entry）
完全にランダムな刺激（単語、画像、概念）を導入し、課題との強制的な接続を試みる。

**手順**:
1. ランダムな名詞を1つ選ぶ（辞書を開く、周囲の物を指す）
2. その名詞の属性をリストアップする
3. 各属性を課題に無理やり結びつける
4. 結びつきから新しい方向性を抽出する

**なぜ機能するか**: 人間の脳はパターン認識に長けており、一見無関係な2つの概念の間にも構造的類似性を見出す。この能力を意図的に発火させる。

### 2. 挑発（Provocation / PO）
「PO」をプレフィックスとして、明らかに非論理的・不可能な statement を作り、そこから有用なアイデアを抽出する。

**手順**:
1. 「PO: [明らかに間違った / 不可能な命題]」を宣言
2. この命題が真だとしたら、何が変わるか考える
3. その変化の中に、実は有用な要素がないか探す

**例**:
- PO: 工場には労働者がいない → 完全自動化の検討
- PO: 顧客が代金を払わない → フリーミアムモデル / 広告モデル
- PO: 問題を解決しない → 問題と共存する戦略

### 3. 反転（Reversal）
問題の方向、因果、主客を反転させる。

**パターン**:
- 主体と客体の反転: 「AがBに作用する」→「BがAに作用する」
- 因果の反転: 「XだからY」→「YだからX」
- 時間の反転: 「まずAしてからB」→「まずBしてからA」
- 目標の反転: 「Xを最大化する」→「Xを最小化するとどうなるか」

### 4. Six Thinking Hats（6つの思考帽子）
意図的に思考モードを切り替える。

| 帽子 | モード | 内容 |
|---|---|---|
| 白 | 事実 | データと情報のみ。判断しない |
| 赤 | 感情 | 直感、感情、好き嫌い。理由不要 |
| 黒 | 批判 | リスク、欠点、なぜうまくいかないか |
| 黄 | 楽観 | 利点、なぜうまくいくか |
| 緑 | 創造 | 新しいアイデア、代替案 |
| 青 | 管理 | プロセスの俯瞰、次のステップ |

## Deep Solver での活用

### Analogist
- ランダムエントリーで、オーケストレータが指定していないドメインからのアナロジーを強制的に探索
- 挑発（PO）で、課題の前提を意図的に破壊して新しい可能性を開く

### Dialectic
- 反転テクニックで Antithesis の方向性を発見
- 「問題を解決しない」という PO から、止揚の手がかりを得る

### Abstractor（メタ層）
- Six Thinking Hats の白帽子→赤帽子の切り替えで、事実分析と感情的直感の両方を活用
- 「この課題を解決する必要があるか」を PO で挑発的に問い直す

### Synthesizer
- 統合時に、Green Hat（創造モード）で、エージェント出力のどれにも含まれない第三の選択肢を探索
