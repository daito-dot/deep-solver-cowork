# Systems Thinking（システム思考）

## 概要
ドネラ・メドウズらが体系化した、相互に結びついた要素からなるシステム全体の振る舞いを理解するための思考法。要素を個別に分析するのではなく、要素間の関係性とフィードバック構造に注目する。

## 核心概念

### ストックとフロー
- **ストック**: 蓄積されるもの（水槽の水量、口座残高、知識量、信頼度）
- **フロー**: ストックを増減させるもの（流入と流出）
- **重要な洞察**: システムの振る舞いはストックの変化で決まり、ストックはフローで変わる。フローを変えないとストックは変わらない。

### フィードバックループ

**正のフィードバック（強化ループ）**: 変化が同方向の変化を加速する
- 例: 人口増 → 出生数増 → 人口さらに増（指数的成長）
- 例: 評判低下 → 顧客離れ → サービス悪化 → 評判さらに低下（死のスパイラル）
- **注意**: 正のフィードバックは常に「成長」ではない。崩壊も加速する。

**負のフィードバック（安定化ループ）**: 変化を打ち消す方向に作用する
- 例: 体温上昇 → 発汗 → 体温低下（ホメオスタシス）
- 例: 在庫増 → 生産減 → 在庫適正化
- **注意**: 安定化は必ずしも良いことではない。悪い状態を安定化するループもある。

### 遅延（Delay）
- フローの変化がストックに影響するまでの時間差
- 遅延が大きいと、オーバーシュート（行き過ぎ）と振動が発生する
- **最も危険なパターン**: 正のフィードバック + 大きな遅延 → 気づいた時には手遅れ

## レバレッジポイント（介入ポイント）

ドネラ・メドウズの12のレバレッジポイント（影響力の低い順から高い順）:

### 低レバレッジ（表層的介入）
12. 定数、パラメータ、数値 — 例: 税率の微調整
11. バッファの大きさ — 例: 在庫量の調整
10. ストックとフローの物理的構造 — 例: インフラの改修

### 中レバレッジ（構造的介入）
9. 遅延の長さ — 例: フィードバックの高速化
8. 負のフィードバックループの強さ — 例: 規制の強化
7. 正のフィードバックループの利得 — 例: 成長率の制御
6. 情報フローの構造 — 例: 誰が何の情報にアクセスできるか

### 高レバレッジ（深層的介入）
5. システムのルール — 例: インセンティブ構造の変更
4. 自己組織化の力 — 例: システムが自ら進化する能力
3. システムの目標 — 例: 何を最適化するかの変更
2. システムのパラダイム — 例: 根本的な世界観の変更
1. パラダイムを超越する力 — 全てのパラダイムが仮説に過ぎないと認識する

## Deep Solver での活用

### Analogist
- 課題のフィードバック構造を特定し、同型のフィードバック構造を持つ他ドメインの事例を探索

### Abstractor（構造層）
- レバレッジポイント分析で「どこに介入すれば最もインパクトが大きいか」を特定
- 具体層の解決策が、低レバレッジポイントへの介入に過ぎないかチェック

### Red Team
- 正のフィードバックループが暴走するシナリオを検証
- 遅延による「手遅れ」シナリオを Pre-mortem で分析

### Orchestrator
- 課題のフィードバック構造の複雑さに基づいて、Cynefin象限を判定
