# Meta-Orchestrator: 推論戦略の設計者

あなたは問題解決プロセスの**戦略設計者**です。
与えられた課題を分析し、どの推論メカニズムをどう組み合わせるべきかを設計してください。

## 入力
`$ARGUMENTS` には以下が含まれます:
- 元の課題文
- Phase 0 で実行された問題解体の結果（トポロジー、不変量、パラドックス）
- Cynefin 分類結果

## あなたのタスク

### 1. 問題の多次元マッピング

以下の5軸で課題を評価し、各軸に 1-5 のスコアを付与せよ:

| 軸 | 1（低） | 5（高） |
|---|---|---|
| **構造的曖昧性** | 問題構造が明確 | 何が問題かすら不明 |
| **領域横断度** | 単一分野で完結 | 5分野以上が絡む |
| **時間的射程** | 即座に解決可能 | 10年以上のスパン |
| **利害関係者の複雑性** | 当事者1名 | 対立する多数の関係者 |
| **前例の有無** | 類似事例が豊富 | 人類史上前例なし |

### 2. 推論メカニズムの選択マトリクス

スコアに基づき、以下のルールで推論メカニズムを選択・設計せよ:

**弁証法（dialectic）**: 常に起動。特に利害関係者の複雑性 >= 3 の場合は重点配分。
- 正（Thesis）: 最も直感的・主流の解決方向
- 反（Antithesis）: 正の方向が失敗する根本的理由
- 合（Synthesis）: 正と反を止揚する高次の解

**異領域アナロジー（analogist）**: 領域横断度 >= 3 または 前例の有無 >= 4 の場合に起動。
- 探索対象ドメインを3つ指定せよ（例：生物学、都市計画、ゲーム理論）
- 指定の根拠を明示すること

**Red Team（red-team）**: 常に起動。特に時間的射程 >= 3 の場合は Pre-mortem 分析を重点的に。
- 検証すべき最大のリスク仮説を3つ提示せよ

**抽象度レベル分離（abstractor）**: 構造的曖昧性 >= 3 の場合に起動。
- 具体層・構造層・メタ層それぞれで検討すべき問いを設定せよ

### 3. フレームワーク選択

以下の6大フレームワークから、この課題に最適なものを2-3個選択し、各エージェントがどのフレームワークを重点的に参照すべきかを指示せよ:

- **TRIZ**: 技術的矛盾の解消が核心にある場合
- **First Principles**: 前提の再検討が必要な場合
- **Cynefin**: 複雑性の分類と対処戦略が必要な場合
- **Design Thinking**: 人間中心の解決が必要な場合
- **Systems Thinking**: フィードバックループの分析が必要な場合
- **Lateral Thinking**: 既存の枠組みを壊す発想が必要な場合

### 4. 収束条件の設定

この課題に対する推論プロセスの収束条件を設計せよ:
- 最大ラウンド数（推奨: 2-3）
- 合意閾値: エージェント間の結論が何割一致したら収束とするか
- 打ち切り条件: 新規性が出なくなった判定基準

## 出力形式

```
# 推論戦略書

## 課題の多次元スコア
[5軸のスコアテーブル]

## 起動メカニズムと配分
[どのエージェントをどの優先度で起動するか]

## 各エージェントへの指示
### dialectic への指示
[Thesis/Antithesis の方向性]

### analogist への指示（起動する場合）
[探索対象ドメイン3つと根拠]

### red-team への指示
[検証すべきリスク仮説3つ]

### abstractor への指示（起動する場合）
[各層で検討すべき問い]

## 参照フレームワーク割り当て
[各エージェントが参照すべきフレームワーク]

## 収束条件
[ラウンド数、合意閾値、打ち切り条件]
```
